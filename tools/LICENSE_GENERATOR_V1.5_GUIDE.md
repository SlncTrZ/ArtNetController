# License Generator V1.5 - User Guide

## 📋 Overview

**Create License for Customer V1.5** is a professional GUI tool for generating cryptographically signed licenses for DMX Master application.

**Version:** 1.5.0  
**Date:** November 4, 2025  
**Status:** ✅ Production Ready

---

## 🔧 Critical Fix in V1.5

### What Changed?

**Previous versions (V1.0 - V1.4)** created licenses with incorrect signature format:
```
Message: device_id + license_id + issued_date  ❌ WRONG
```

**V1.5** uses the CORRECT format:
```
Message: device_id|license_id|issued_date  ✅ CORRECT
```

This matches the verification logic in `src/utils/license.py` line 377-378.

### Why This Matters

- ✅ Licenses generated by V1.5 will activate successfully
- ❌ Licenses from older versions will show "Invalid license key or device mismatch"
- ⚠️  Always use V1.5 or later to generate customer licenses

## 🚀 How to Use

### Step 1: Get Customer Device ID

Ask customer to:
1. Run DMX Master application
2. Go to Settings → License
3. Copy the **64-character Device ID** (hex string)

Example Device ID:
```
226e6b3a1bf085b0ba538f7139d210cfbcf2acfe6d434079d39da1f06b1b3775
```

### Step 2: Open License Generator

Run: `Create License for Customer V1.5.exe`

### Step 3: Fill Customer Information

1. **Device ID:** Paste the 64-character hex string
   - Tool will validate automatically
   - ✅ Green checkmark = Valid
   - ❌ Red error = Invalid format

2. **Customer Email:** Enter customer's email address
   - Example: `customer@example.com`

3. **License Type:** Select from dropdown
   - `perpetual` - Never expires (recommended)
   - `trial` - Time-limited trial
   - `subscription` - Renewable subscription

### Step 4: Generate License

Click **🎯 Generate License** button

The tool will:
- Create a unique License ID (UUID)
- Generate cryptographic signature (RSA-2048 PSS)
- Display JSON license data

### Step 5: Send to Customer

Choose one of:

**Option A: Copy to Clipboard**
1. Click **📋 Copy to Clipboard**
2. Paste into email/chat and send to customer

**Option B: Save to File**
1. Click **💾 Save to File**
2. Choose location (suggested: `license_DEVICEID.json`)
3. Send file to customer

## 🔍 Troubleshooting

### "Invalid license key or device mismatch"

**Cause:** License was generated for different Device ID

**Solution:**
1. Ask customer to send their CURRENT Device ID
2. Verify the Device ID matches exactly (case-insensitive)
3. Generate a NEW license with correct Device ID

### "Invalid license format"

**Cause:** JSON is corrupted or incomplete

**Solution:**
1. Copy the ENTIRE JSON including `{` and `}`
2. Make sure all quotes are preserved
3. Re-send the license

### Tool Crashes When Pasting Device ID

**Cause:** You're using an older version with bugs

**Solution:**
1. Use V1.5 or later
2. Make sure you're pasting ONLY the hex string (no spaces, no quotes)
3. Device ID must be exactly 64 characters

## 📁 Files Generated

Each license creates:
- **License ID:** Unique UUID (e.g., `406c9541-649c-419a-87e3-6c61385c6065`)
- **Signature:** Base64-encoded RSA signature (256 bytes)
- **Issued Date:** ISO timestamp (e.g., `2025-11-04T16:54:08.980754`)
